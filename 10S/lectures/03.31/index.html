  • arrays

Basics of using Ch

There is a very useful program on CSIL called Ch that you can use to learn about some of the basic properties of the C programming language.

Ch is an 'interpreter' for C—it allows you to type in C statements one at a time, and have those statements be immediately turned into something that can be executed by the computer, one at a time. This is really helpful for trying out small bits of code to see what they will do.

Ch has many other uses, as we may see later in the course—but for today we'll primarily be using it for little experiments to learn about:

  • declaring variables
  • assignments statements
  • the printf command.

To bring up Ch on CSIL, login in, then simply type ch at the command prompt.

Basics of C

In C, every variable has to be declared the first time it appears.

Declaring variables: declaring a variable assigns it a name and a type.
The declarations below also allocate memory for the variables that are declared.

The part that appears after the // is a comment—it is there only for the human reader of a C program.

int x; // x can store an integer value like -42, 3 or 16;
double y; // y can store a real number like 3.14, -0.00001, or 6.0221415E23

char c; // c can store a single character like 'A', '3', or '?'

We can try these at the Ch prompt—to make the typing easier, we won't include the comments:

Ch> int x;
Ch> double y;
Ch> char c;
Ch>

Note that:

  • each of these declarations ends in a semicolon.
  • the proper names in C are int, not integer, and char, not character.
  • although there is a type called float for "floating-point numbers" we hardly ever use it
    • we use double instead, which works like float, but uses 2x the memory, so its more accurate
    • double is short for double-precision floating point
    • back in the day, memory was expensive and the CPU was slow—so using double was a luxury
    • nowadays, memory is cheaper, and computers are faster: we almost always use double.
  • In C, for true/false values, we generally just use int, and let 0 stand for false, and 1 for true.
    • There is usually no separate boolean type in C
    • There is a boolean type in C++/Java, but that's for later courses (CS24,CS32,CS56)
  • There are many other types, but int, double, char is enough to get started.

Initializing variables: initializing a variable gives it an initial value. 

x = 4;
y = 4.5;
c = 'A'; // single characters must be in single quotes

At the Ch prompt, we can make these assignment statements, and then type just the name of a variable to see what was assigned. Note that y is printed with a few extra zeros—we'll discuss later how we can modify the way it gets printed.

Ch> x = 4;
4 
Ch> x
4 
Ch> y = 4.5;
4.5000 
Ch> y
4.5000 
Ch> c = 'A';
A 
Ch> c
A 
Ch>

Combining declaration and initialization: Declaration and initialization can be combined:

int t=4;
double u=5.6;
char v='B';

This is a way to make your code more compact. It also allows you to make sure that variables have an initial value—if you don't do it, the initial value is undefined, and could be anything! That's leads us to a discussion of "junk data" (see below).

Try these in Ch:

Ch> int t=4;
Ch> double u=5.6;
Ch> char v='B';
Ch> t
4 
Ch> u
5.6000 
Ch> v
B 
Ch>

Note that if you try to redeclare a variable that has already been declared, you'll get an error, as the example below shows. After writing int w=4; you can write an assignment statemet such as w=5; to change the value of w, but you should not put the word int in front of w the second time:

  • You can use an assignment statement on a variable as many times as you want to give it a new value
  • But you can only declare a variable once to give it a type.
    • The actual rule is "once per scope"—but that's getting ahead of ourselves—we'll talk about scope later.

Ch> int w=4;
Ch> int w=5;
ERROR: identifier 'w' redeclared
Ch> w
4 
Ch> w=5;
5 
Ch>

So, in C, the type of a variable is permanent, and established when the variable is declared—e.g. once w is an int, it is always an int—but its value may be changed, for example by an assignment statement.

Try these things:

  1. What if I declare int a = 5; and then try int a = 5; (the same declaration again)
    • Is the result the same or different from the int w=4; then int w=5; case?
  2. What if I declare int b = 5; and then try double b = 5.0; (same variable but different type)?
    • Is the result the same or different from previous cases?
