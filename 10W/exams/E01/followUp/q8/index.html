There are many ways to answer this question correctly—here's an analysis of the correct answers I got:

Correct Answer Group 1: (24/72 responses, ~33%)
Answers based on counting up

What all of the answers in this group have in common is that they all count up (the for loop ends in i++),
and they all use either printKSpacesNStars(i, width-i); or some slight variation on that.

The most common answer was this one.

Correct Answer 1a (20/72 responses)

for (i=0; i<height; i++)
{
  // print i spaces then width-i stars and a newline
  printKSpacesNStars(i, width-i); printf("\n");
}

Considering this as a change from the sample code given, to arrive at this answer, you had to reverse the parameters to printKSpacesNStars, and change the for loop to count from 0 to height-1 instead of from 1 to height.

An slight variation on that answer was this one: the same but with width instead of height in the for loop:

Correct Answer 1b (2/72 responses):

for (i=0; i<width; i++)
{
  // print i spaces then width-i stars and a newline
  printKSpacesNStars(i, width-i); printf("\n");
}

Another slight variation was to use a <= operator, in which case it is necessary to adjust the right operand:

Correct Answer 1c (1/72 responses):

for (i=0; i<=height-1; i++)
{
  // print i spaces then width-i stars and a newline
  printKSpacesNStars(i, width-i); printf("\n");
}

And finally in this group, it is also possible to count from 1 to height, but then you must adjust the parameters to the function by subtracting 1 from i:

Correct Answer 1d (1/72 responses):

for (i=1; i<=height; i++)
{
  // print i spaces then width-i stars and a newline
  printKSpacesNStars(i-1, width-(i-1)); printf("\n");
}

Correct Answer Group 2: (15/72 responses, ~21%)
Answers based on counting down

What all of the answers in this group have in common is that they all count up (the for loop ends in i-),
and they all use either printKSpacesNStars(width-i, i); or some slight variation on that.

The most common answer in this group was:

Correct Answer 2a (7/72 responses)

for (i=height; i>=1; i--)
{
  // print width-i spaces then i stars and a newline
  printKSpacesNStars(width-i,i); printf("\n");
}

Two slight variations on this answer changed either the termination condition to an equivalent one, and used either height or width as the initialization for i:

Correct Answer 2b (4/72 responses)

for (i=height; i>0; i--)
{
  // print width-i spaces then i stars and a newline
  printKSpacesNStars(width-i,i); printf("\n");
}

Correct Answer 2c (2/72 responses)

for (i=width; i>0; i--)
{
  // print width-i spaces then i stars and a newline
  printKSpacesNStars(width-i,i); printf("\n");
}

There were two more counting down answers that were technically correct, though needlessly complicated:

Correct Answer 2d (1/72 responses)

for (i=width; 0<i; i--)
{
  if (i==width)
     printNChars(width, '*'); printf("\n");
  else
    printKSpacesNStars(width-i,i); printf("\n");
}
The 0 < i isn't a problem here—that's a style preference that some coders prefer because the natural order of left-to-right, smallest to largest. The problem is that the code inside the if is actually equivalent to the code inside the else when the if condition is met. Thus, there is really no need to pull this out as a special case.

Correct Answer 2e (1/72 responses)

This next correct answer is one that at first looks like a logic error, since the for loop appears to be an infinite loop:

for (i=height; i<=height; i--)
{
  if (i==0)
  {
     return 0;
  }
  printKSpacesNStars(width-i,i); printf("\n");
}

Since i starts at height and goes down, the condition i<=height is true the first time, and will remain true forever (or at least until i wraps around past the most negative integer.)

But because there is an if inside the loop that checks for i==0 and does a return to end the function, the loop actually does terminate.

It would be much simpler to just write this like answer 2b, to which it is, in effect, equivalent (provided there is no code after the for loop inside the function.)

Correct responses with minor syntax errors (4/72, ~6%)

There were four more papers with correct answers that had minor syntax errors.

I deducted -1 for using , instead of ; in the for loop:

Wrong: for (i=0, i<height, i++)
Correct: for (i=0; i<height; i++)

I deducted -1 for the wrong slash in the newline character:

Wrong: printf("/n");
Correct: printf("\n");

I deducted -2 for not putting braces on the body of the for loop. In Python, it is sufficient to indent code to show grouping under a for loop, but in C, if there is more than one statement, the braces are required. This is even true if the statements are on a single line separated by semicolons:

Wrong

for (i-0; i<width; i++)
    printKSpacesNStars(i,width-i); 
    printf("\n");

Correct:

for (i-0; i<width; i++)
{
    printKSpacesNStars(i,width-i); 
    printf("\n");
}

Minor Logic Errors (12/72, ~17%)

There were two minor logic errors for which I assigned deductons of -2 in each case.

12 of the 72 papers contained only one or both of these minor logic errors, but were otherwise ok.

Because some of the papers contained the minor logic errors below in combination with each other, or with one or more of the more serious logic errors, the totals for the subcategories may sum to more than 72.

Minor logic error 1: < vs <= (12/72, ~17%)

This logic error had different forms based on which of the correct answers it occured in the context of.

If we look at it in the context of correct answer 1a, it looks like this:

for (i=0; i<=height; i++)
{
  // print i spaces then width-i stars and a newline
  printKSpacesNStars(i, width-i); printf("\n");
}

Because <= was used instead of <, we get one extra row. It so happens that the extra row is a row of blanks, but it is still one too many rows in the output.

The breakdown of students making this error was:

  • 9/72: similar to correct answer 1a, but used for (i=0; i<=height; i++)
  • 1/72: similar to correct answer 1b, but used for (i=0; i<=width; i++)
  • 2/72: similar to correct answer 2c, but used for (i=width; i>=0; i--)

{

What kind of pattern would that produce? (This is left as an exercise for you)

Version 1b: If the loop counts from 1 to height
then the parameters to printKSpacesNStars must be (i-1, width-i+1)

{

Serious logic error 4:premature return (-4)

At least one paper contained a premature return. The following answer is completely correct except that it has a return statement inside the loop. If this return statement were removed, or moved outside the loop, this would be an example of correct answer number 2b:

for (i=height; i>0; i--)
{
  // print width-i spaces then i stars and a newline
  printKSpacesNStars(width-i,i); printf("\n");
  return;
}

As an exercise: what would this produce for height and width of 4, 5 or 6? And why?

Serious logic error 5: confusing printKSpacesNChars with printNChars (-4)

The following answer is one that illustrates some of the strangeness that can result from the way that C conflates int with char. The following answer would likely compile and run but it could produce strange (and incorrect) results.

for (i=height; i>0; i--)
{
  int j;
  j = height -i;
  printNChars(j,i);
  printf("\n");
}

This answer would be a correct variation on correct answer 2b if the person had written, instead:

for (i=height; i>0; i--)
{
  int j;
  j = height -i;
  printKSpacesNChars(j,i);
  printf("\n");
}

In that case, this would just been a variation where a separate variable j is used instead of writing height -i for the first parameter. However, instead, the person used printNChars(j, i); As a result, this would print the character referenced by i, and it would print that character j times.

You might thing, therefore, that for input of 5, this would print the following—since:

  • the first time through the loop, j=0, i=5
  • the second time, j=1, i=4
  • the third time, j=2, i=3
  • etc.
what we want what you might think we would get 
*****
 ****
  ***
   **
    *


4
33
2222
11111

However, that misses a key point: the second parameter is being passed in as an int, but is being treated in the function as a char. That means that instead of a triangle (of the wrong shape) of numbers, we'll get a triangle composed of the ASCII characters 4, 3, 2 and 1, all of which are "control characters" that won't show up on your screen as anything you can see. So, essentially you'll get what appears to be no output at all.

Serious logic error 7: (difficult to describe) (-8)

One paper was a mixture of several problems.

for (i=0, i< height; i++)
for (i=1, i<=height, i++)
{
  printKSpacesNStars(i, width-1); 
  printf("\n");
}

There are a variety of problems here:

  • Nested for loops where a nested for loop is not needed—the printKSpacesNStars function already abstracts away the "inner for loop" that prints all the characters along a row.
  • Nested for loops that use the same variable, in this case i. If you have nested for loops, the two loops need different variables (e.g. i and j)
  • Syntax issues with , vs ; in the for loops.

Given that the parameters to printKSpacesNStars are (i, width -i), the correct for loop is a single for loop—any of the ones from correct answers 1a, 1b, or 1c, e.g. for (i=0; i<height; i++)
