  1. (10 pts) Given the definition of a struct for Point:

    struct Assignment {
      int pointValue; // how many pts the assignment was worth
      int pointsEarned; // how many pts the student earned on this assignment
    };

    Write the definition of a function that matches this prototype:

    double pctScoreForAssignment(struct Assignment *scores,
                             int assignmentNumber)
                             

    This function takes an array of struct Assignment entries called scores, and assignmentNumber which is an index into that array.

    The function should return the score on the assignment that is at index assignmentNumber, as a percent. That is, you should divide the points earned by the point value, and multiply by 100. Keep in mind that the result might not be an integer.






  2. Here is a recursive function with the name mysteryFunction. As parameters, it takes an array of integers a, and the length of that array n as parameters, and returns an integer. What it actually computes is a mystery for you to solve.

    int mysteryFunction(int *a, int n)
{
  if (n<=0)
    return 0;
  else
    return a[0] + mysteryFunction(a+1,n-1);
}
    1. (5 pts) Solve the mystery by figuring out what the mystery function computes, then come up with a more reasonable name for the function.

      Hint: Among these possible function names, one (and only one) of them would be a reasonable answer to this question: average, countEven, countMax, countMin, countNeg, countOdd, countPos, countSevens, indexFirstEven, indexFirstOdd, indexOfMax, indexOfMin, isSorted, maxValue, minValue, noDups, sum.

    2. (10 pts) Rewrite the function so that it computes the same result for every input, but using iteration (a for loop or while loop) instead of recursion.
